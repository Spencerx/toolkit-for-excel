Event Handling


Events are captured in the Word application and information is passed to MarkLogicContentControlSupport.js. To insure framework code is not tightly coupled with application code, developers should define their custom content control event handler functions in a separate .js file and call from MarkLogicContentControlSupport.js.

Following events captured:

contentControlAfterAdd()
contentControlBeforeContentUpdate()
contentControlBeforeDelete()
contentControlBeforeStoreUpdate()
contentControlOnEnter()
contentControlOnExit()

Event Detail

contentControlAfterAdd()
    Occurs after adding a content control to a document. Creates a MLA.SimpleContentControl() to forward to a custom handler.
contentControlBeforeContentUpdate()
    Occurs before updating the content in a content control, only when the content comes from a mapped custom XML part. Creates a MLA.SimpleContentControl() to forward to a custom handler.
contentControlBeforeDelete()
    Occurs before removing a content control from a document. Creates a MLA.SimpleContentControl() to forward to a custom handler.
contentControlBeforeStoreUpdate()
    Occurs before updating the custom XML part the content control in the document is mapped to with the value of a content control. Creates a MLA.SimpleContentControl() to forward to a custom handler.
contentControlOnEnter()
    Occurs when a user enters a content control. Creates a MLA.SimpleContentControl() to forward to a custom handler.
contentControlOnExit()
    Occurs when a user leaves a content control. Creates a MLA.SimpleContentControl() to forward to a custom handler.
